# Rams

> *"Weniger, aber besser." — Less, but better.*
> — Dieter Rams

## Method: pay-rent test

Subtractive. Every element must earn its presence. Start with the content stripped to essence; add only what demonstrably helps.

Rams asks, for every element on the page:

**"Does this earn its rent?"**

If removing it would not demonstrably worsen the user's understanding, remove it.

## When to select Rams

Use Rams when:

- The subject is functional — a product, a tool, a service, a dashboard
- The audience came to DO something, not to FEEL something
- Honesty outranks expression ("this is a calculator; let it be a calculator")
- The subject's value is in *what it does*, not *what it means*

Do NOT use Rams when:

- The subject is cultural, emotional, or expressive (use Saville, Scher, or Weingart)
- The subject needs to move the reader (Rams moves nobody)
- The subject depends on atmosphere (Rams kills atmosphere)

## Method — step by step

### Step 1. State the purpose in one sentence

*"This page exists so the user can X."* Write it down. Every element on the page must serve X or be removed.

### Step 2. Strip

Start with the leanest possible layout: content, hierarchy, and nothing else. No decoration. No hero image. No ambient motion.

### Step 3. Add only what earns rent

Test each candidate addition:
- Does this help the user DO the thing faster, better, more clearly?
- Or is this there because you're afraid of empty space?

If the second, remove.

### Step 4. The honesty test

Every remaining element should be *obvious in purpose*. A button looks like a button. A link looks like a link. A form field looks like a form field.

Rams is NEVER clever. Clever is a tax on comprehension.

## Aesthetic posture

- **Palette**: monochromatic or near-monochromatic. One accent at most, used only for interactive states. Rams's Braun products are famous for their restraint — one color did the entire job.
- **Typography**: one face. Neutral, industrial, readable. Helvetica, Inter, Söhne, Diatype. Size scale tight — maybe 3 sizes total.
- **Motion**: **reactive only** (per `principles/motion.md` category 4). Motion provides feedback, signals state, nothing else. No ambient. No punctuating.
- **Microfeature**: **none**. The discipline is the craft. Decoration violates the method.
- **Imagery**: product-honest. White-ground industrial photography. No atmosphere. No people gazing into middle distance.
- **Whitespace**: generous but not theatrical. Whitespace as clarity, not as pressure.

## Anti-patterns

- **Hero decorations** (abstract gradients, ambient particles, hero illustrations — all violations)
- **Multiple accent colors** (Rams uses zero or one)
- **Punctuating motion** (reveals, scroll effects, cursor flourishes — all violations)
- **Hero imagery of people using the product** (Apple-style lifestyle photography is not Rams)
- **Clever microcopy** (Rams is direct. "Submit" not "Let's go!")

## The 10 principles (Rams's own)

1. Good design is innovative
2. Good design makes a product useful
3. Good design is aesthetic
4. Good design makes a product understandable
5. Good design is unobtrusive
6. Good design is honest
7. Good design is long-lasting
8. Good design is thorough down to the last detail
9. Good design is environmentally friendly
10. Good design is as little design as possible

All ten are tests you can apply to your output. If your page fails #5 (unobtrusive), #6 (honest), or #10 (as little as possible), regenerate.

## Rams's question

Before writing any CSS, Rams asks:

> **"What can I remove without making this worse?"**

Remove it. Ask again. Repeat until removal would break the page.
